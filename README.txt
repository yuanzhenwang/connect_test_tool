命令使用说明：

	缺省模式  当不给工具指定运行命令时，默认开启自动连接模式，此时连接配置文件中指定类型的设备；
			 开启此模式之前必须在配置文件在指定要连接的设备名称
			 该模式会无限循环连接设备，必须关闭dos窗口才能结束程序

	connt    使用该命令之前必须在配置文件中指定要连接的设备，命令后无需任何参数

	dis_one  断连一个设备，需要在命令后面指定断连的设备mac

	dis_all  断连所有设备，后面无需任何参数

	get      获取所有连接设备，后面无需任何参数

	auto	 自动扫描一段时间统计扫描结果，然后开始连接设备。后面可选参数为结果目录，默认为logs目录


help命令：

	工具内置help命令，使用时可以用help参数查看。
	用法：
	python 脚本.py -help/--help/help


配置文件说明：
	
	注意：所有的字段值修改必须在英文输入状态下修改！

	"devices":"['24:71:89:E8:8D:07','24:71:89:B0:15:AB','CC:1B:E0:E8:08:16','FC:7A:15:41:2D:AC','E4:55:7E:48:08:BC']"
	该参数指定需要连接的设备，格式必须为上面所示，如果想添加设备，按照格式扩展即可；
	该字段的参数在conn命令执行时才会生效，如果不运行conn命令该字段无效

	"device_type_name":"['CC2650 SensorTag','S1000']"
	该字段指定需要连接的设备名称，扩展方式跟device相同；
	该参数在不给程序指定要执行的命令的时候执行，指定执行命令的时候该字段无效

	"host":"http://172.16.40.200/api"
	该字段指定AC的地址，当使用本地API时，配置方式如下："host":"http://172.16.40.200"，即指定AP的IP地址即可
	如果采用本地API，必须将下面的local参数设设为True

	"user":"tester"
	"pwd":"10b83f9a2e823c47"
	开发者账号和密码，此字段在本地API时无效

	"ap":"CC:1B:E0:E0:EB:3C"
	AP的MAC地址，用来指定所用的AP，当调用本地API时该字段无效

	"chip":0 or 1
	配置扫描用的芯片

	'scan_mode':0 or 1
	配置扫描模式，0代表被动，1代表主动

	"local_host":http://AP IP address
	配置本地扫扫描使用的IP地址

	"local":0 | 1
	用来标识是用本地API还是云API，1用本地API，0用云端API

	"rssi_count":10
	设置开始连接设备收集多少个RSSI计算RSSI平均值

	"retry":3
	自动连接状态下，设置最大重连次数，超过最大次数后放弃本次连接。
	默认重连次数为5
